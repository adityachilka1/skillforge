# Contributing

Thanks for considering a contribution. This guide is short on purpose.

## Before you start

- Search the existing [issues](../../issues) and [pull requests](../../pulls) to avoid duplicate work.
- For non-trivial changes, open an issue first so we can talk about scope before you write code.
- By contributing you agree your work is released under the project's existing license.

## Pull request checklist

- One purpose per PR. Smaller PRs review and merge faster.
- Reference the related issue (`Fixes #N` / `Closes #N`) in the PR body.
- Update docs and tests for behaviour changes.
- Run the project's own check command before pushing (CI runs the same one).
- Use [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) style for commit titles when the project uses them.

## Reviewing

Anyone is welcome to review. Helpful comments are specific, kind, and grounded in the actual diff.

## Code of Conduct

See [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md). In short: be kind, assume good faith.
